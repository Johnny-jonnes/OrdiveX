1. Prérequis & Matériel

OrdiveX est une PWA (Progressive Web App) ultra-légère. Elle ne nécessite aucun serveur local, mais demande un navigateur moderne pour fonctionner de manière optimale.

Terminaux

PC (Windows/Mac), Tablettes (iPad/Android) ou Smartphones. Un écran de 10 pouces minimum est recommandé pour le point de vente.

Connectivité

Internet requis pour l'installation et la synchronisation. L'application reste utilisable 100% hors-ligne pour les ventes.

2. Installation PWA

L'installation transforme le site web en une application native sur l'appareil, supprimant la barre d'adresse et permettant l'accès hors-ligne.

Sur Android (Chrome)

  1. Accédez à l'URL de votre instance OrdiveX.
  2. Attendez l'apparition du bandeau en bas de l'écran ou allez dans le menu ⋮.
  3. Sélectionnez "Installer l'application".

Sur iOS (Safari)

  1. Appuyez sur le bouton Partager (carré avec flèche).
  2. Faites défiler vers le bas et appuyez sur "Sur l'écran d'accueil".
  3. Validez le nom de l'application.
Important : Une fois installée, lancez toujours l'application depuis son icône sur l'écran d'accueil pour garantir la stabilité et la persistance des données localement.

3. Onboarding & Personnalisation

Au premier lancement, l'assistant de configuration définit l'identité visuelle de la pharmacie.

Identité Visuelle

Téléchargez le logo de la pharmacie. Il sera utilisé sur tous les tickets de caisse et factures A4 générés.

Coordonnées

Le nom, l'adresse et le téléphone sont obligatoires pour la conformité des documents légaux.

Option de Restauration : Si vous installez un nouveau terminal pour une pharmacie existante, utilisez le Magic Link lors de l'onboarding pour récupérer instantanément toute la configuration.

4. Configuration Cloud (Supabase)

Indispensable pour la synchronisation multi-terminaux et la sauvegarde automatique en temps réel.

Étape 1 : Création du projet

Créez un compte gratuit sur supabase.com et créez un nouveau projet "Pharmacie [Nom]".

Étape 2 : Initialisation SQL

Copiez le contenu du fichier supabase_schema.sql dans le SQL Editor de Supabase et cliquez sur Run.

Étape 3 : Liaison API

Dans OrdiveX, allez dans Paramètres > Synchronisation Cloud et renseignez :

  • Supabase URL
  • Supabase Anon Key

5. Import de Données & Catalogue

Le succès du déploiement repose sur la qualité du catalogue initial. Voici comment procéder selon votre situation.

Scénario A Pharmacie sur Cahier

Aucune donnée numérique n'est disponible.

  • Méthode : Saisie manuelle progressive via Produits > Ajouter.
  • Conseil : Priorisez les 20% des produits qui génèrent 80% des ventes (Loi de Pareto).
Scénario B Fichier Excel/CSV

Vous disposez déjà d'une liste sous Excel.

  • Action : Adaptez votre tableau aux colonnes OrdiveX : Code, Nom, DCI, Marque, Categorie, Prix Vente, Prix Achat, Rx.
  • Import : Allez dans Produits > Importer et glissez votre fichier CSV (UTF-8).
Scénario C Logiciel Existant

Migration depuis un ancien système (ex: logiciel sous Windows).

  1. Exportez le catalogue au format Excel/CSV depuis l'ancien logiciel.
  2. Utilisez le script de mapping pour aligner les colonnes (Désignation → Nom, PA → Prix Achat, etc.).
  3. Nettoyez les doublons avant l'import final.
Fichiers volumineux : Pour les catalogues de plus de 50 000 références, l'import peut prendre plusieurs minutes. L'application traite les données par lots de 1000 pour ne pas ralentir l'interface.

7. Paramétrage Assurances

OrdiveX gère nativement le tiers-payant pour les entreprises et mutuelles de Guinée.

  1. Lors d'une vente, sélectionnez le mode "Assurance".
  2. Renseignez le nom de la mutuelle (ex: NSIA, SONAM, etc.).
  3. Saisissez le montant ou le pourcentage pris en charge.
  4. Le système sépare automatiquement la part patient de la part entreprise dans les rapports.

8. Initialisation du Stock

Une fois le catalogue importé (Quantité = 0), vous devez initialiser votre inventaire réel.

  • Option Rapide : Ajustement manuel via Stock > Entrée de stock.
  • Option Pro : Création d'un Bon de commande fournisseur, réceptionné pour alimenter le stock avec traçabilité des lots et dates de péremption.

9. Checklist Finale

  • PWA installée sur l'écran d'accueil
  • Logo et coordonnées vérifiés sur les factures
  • Test de synchronisation Cloud réussi
  • Profils caissiers créés et testés
  • Sauvegarde manuelle effectuée (export JSON)

10. Dépannage & Support

Écran Blanc

Videz le cache du navigateur ou vérifiez la connexion internet lors du premier chargement.

Erreur Sync

Vérifiez les clés API Supabase dans les paramètres et assurez-vous que le RLS est désactivé sur les tables.
